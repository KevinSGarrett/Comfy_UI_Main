# Wave70 Model-Backed Geometry Authority

## Purpose

Wave70 needs a local-first autonomous geometry authority before mask generation. The current failure mode is clear: Haar, Canny, broad rectangles, and hand-tuned one-image coordinates are diagnostic aids, not anatomy. They cannot reliably locate occluded eyes, eyelids, lashes, nose sidewalls, lips, cheeks, jaw/chin, neck, hairline, hands, clothing, contact patches, or body regions.

This plan defines the implementation work required to replace shape guessing with source-derived landmarks, semantic parsing, promptable segmentation refinement, confidence scoring, and canonical polygon export.

## Authority Pipeline

Required flow:

```text
source image
  -> dependency/model probe
  -> landmarks / pose / hand anchors
  -> semantic parsing
  -> promptable segmentation refinement
  -> visibility and occlusion scoring
  -> model consensus metrics
  -> canonical source-derived polygons
  -> geometry hard gate
  -> mask generation
  -> semantic mask QA
  -> generated-output QA
```

## Implementation Requirements

### 1. Dependency And Model Probe

Autonomously probe for local Python/runtime support, model files, and adapters:

- OpenCV and image I/O
- MediaPipe-compatible face/pose/hand landmark route
- torch-compatible segmentation route when needed
- face parsing model route
- SAM/SAM2-compatible promptable segmentation route
- fallback blocker writer

If a dependency or model is missing, write a blocker; do not ask the user for manual installation during normal execution.

### 2. Face Landmark Authority

Implement a face landmark adapter that outputs source-space anchors for:

- face oval and jaw/chin contour
- hairline/forehead boundary candidates
- left/right eye aperture
- upper/lower eyelid folds
- eyelash line candidates when visible
- eyebrow hair bands
- nose bridge, sidewalls, tip, nostrils, alae
- upper/lower lip and mouth corners
- neck/chin separation where visible

The adapter must distinguish subject-left/right from viewer-left/right and record both labels.

### 3. Semantic Face Parsing Authority

Implement or block a face parsing adapter that outputs semantic regions for:

- skin/face
- hair
- brows
- eyes
- nose
- upper lip
- lower lip
- mouth/teeth when visible
- neck
- clothing/background separation when possible

Parsing output must be checked against landmarks and user/gold traces.

### 4. Promptable Segmentation Refinement

Implement or block a SAM/SAM2-compatible refinement adapter. It must use:

- positive points from target landmarks/parsing
- negative points from protected neighbors
- bounding prompts only when supported by fine source-derived prompts
- mask stability score or equivalent confidence

SAM refinement cannot invent occluded anatomy. If target visibility is insufficient, block.

### 5. Visibility/Occlusion Confidence

Implement a visibility resolver that classifies each target as:

```text
visible
partially_visible
occluded
not_visible
too_low_resolution
model_disagreement
```

Occluded or ambiguous regions must not be generated by symmetry. For the current portrait, the viewer-left eye/hair overlap is a known hard case.

### 6. Gold Trace Dataset And Metrics

Use existing user-provided reference annotations as a gold calibration set. Store them as structured source-space polygons with:

- image hash
- region labels
- polygon points
- viewer-left/subject-left mapping
- visibility state
- protected-neighbor relationships

Compare generated geometry against gold traces using IoU, boundary distance, center drift, protected overlap, and confidence. Gold traces are test evidence, not future manual work.

### 7. Consensus Validator

Implement a model consensus validator. It must block when landmarks, parsing, refinement, and gold/reference traces disagree beyond declared thresholds. It must emit JSON evidence and a readable panel.

### 8. Canonical Polygon Export

Only source-derived, consensus-passed polygons or segmentation maps can become canonical geometry for mask generation. Debug rectangles, Canny edges, Haar boxes, and one-image coordinates are never canonical by themselves.

### 9. Body, Hand, Clothing, Contact Extension

Generalize the same authority pattern through `Plan/Instructions/QA/WHOLE_BODY_GEOMETRY_AUTHORITY_PROTOCOL.md` and `Plan/07_IMPLEMENTATION/mask_factory/WAVE70_WHOLE_BODY_GEOMETRY_AUTHORITY.md`.

- pose landmarks for torso/limbs/joints
- hand landmarks for hands/fingers/fingernails
- segmentation for clothing/background/hair/skin
- contact patches from source-visible geometry and protected anchors
- temporal consistency for video masks
- body region geometry only behind explicit body region geometry, source/context, and explicit route gates

Existing hand, hand-interaction, contact, support, body, and soft-body masks are not trusted until this whole-body authority regenerates or blocks them from canonical source-derived geometry.

### 10. Reference Matrix Validation

Generalized claims require reference-image matrix validation across subjects, skin tones, angles, expressions, occlusion, resolution, and visibility cases.
