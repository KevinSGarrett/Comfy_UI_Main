# Wave74 Autonomous Simulation Backend Adapters

Purpose: Define the adapter layer that can call Blender first, then optional Houdini, Unreal, DAZ, Marvelous/CLO, or ComfyUI-only approximation routes without manual work or drift.

Activation gate: Deferred. Adapter implementation starts only after the schema and production-base requirements are stable and the current ComfyUI work is not blocked by this future layer.

This wave is intentionally deferred. It is included so the autonomous project has exact end-to-end planning, Items, Tracker rows, QA gates, and future implementation instructions. It must not displace nearer active ComfyUI runtime, workflow, cost-control, or current-lane work.

Autonomy rule: after any user-supplied prototype/reference asset is registered, all setup, conversion, adapter execution, map generation, routing, testing, QA, visual review, video review, audio review, and blocker recording must be autonomous.

Strict QA rule: no row can be promoted from documentation, taxonomy presence, a single smoke artifact, or target-region-only review. Generated outputs require whole-artifact visual review and, for video/audio work, full-duration temporal/audio review.

External automation references:
- Blender command line and background automation: https://docs.blender.org/manual/en/latest/advanced/command_line/arguments.html -- Blender-first adapter can run scripted scene import, fitting, simulation prep, render-pass export, and validation from command line.
- SideFX Houdini hython/hbatch batch workflow: https://www.sidefx.com/docs/houdini/render/batch.html -- Optional Houdini adapter can run procedural/SDF/Vellum-style simulation and render/export tasks through command-line tools.
- Unreal Python command-line scripting: https://dev.epicgames.com/documentation/unreal-engine/scripting-the-unreal-editor-using-python?lang=en-US -- Optional Unreal adapter can automate editor/script tasks for physics assets, animation, rendered passes, and real-time simulation evidence.
- DAZ Studio scripting: https://docs.daz3d.com/public/software/dazstudio/4/referenceguide/scripting/start -- DAZ is treated as prototype sculpt/source intake and scriptable export surface, not the production physics authority.
- DAZ silent FBX export sample: https://docs.daz3d.com/public/software/dazstudio/4/referenceguide/scripting/api_reference/samples/file_io/export_fbx_silent/start -- DAZ prototype packages can be exported without interactive export dialogs when DAZ is available.
- DAZ neutral prototype boundary: https://docs.daz3d.com/public/software/dazstudio/4/referenceguide/scripting/start -- DAZ is a neutral A/T-pose prototype and export source only; DAZ dForce, grip posing, hand posing, runtime morphing, production rigging, and production deformation are not allowed after prototype registration.
- Marvelous Designer Python API: https://developer.marvelousdesigner.com/ -- Optional clothing adapter can automate garment import, simulation, and export where licensed/available.
- Marvelous Designer export API list: https://developer.marvelousdesigner.com/list.html -- Optional clothing adapter can export OBJ/Alembic and related artifacts for downstream fitting, collision, and map baking.
