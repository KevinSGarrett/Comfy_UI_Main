# Audio Generation Review Protocol

## Purpose

This protocol defines autonomous QA for speech, music, ambience, and mixed audio outputs.

## Required review areas

Codex must review:

- voice clarity
- noise
- clipping
- distortion
- pacing
- emotional tone
- pronunciation
- unnatural pauses
- robotic artifacts
- music / voice balance
- stereo / mono issues
- loudness consistency
- synchronization with video if applicable
- generated speech correctness
- unwanted background sound

## Review flow

1. Confirm file integrity and successful playback.
2. Review beginning, middle, and end of the file.
3. Review loud passages, quiet passages, and transitions.
4. Compare transcript or expected text against actual spoken output when applicable.
5. If paired with video, check sync.
6. Record defects and severity.

## Scoring categories

- intelligibility
- cleanliness
- emotional / stylistic fit
- technical consistency
- content correctness
- sync quality if applicable

0–5 scale per category.

## Blocking defects

- unintelligible speech
- major clipping or distortion
- incorrect spoken content in a critical task
- severe robotic or glitch artifacts
- loudness jumps that make the file unfit for use
- significant sync drift when paired with video

## Evidence required

- audio QA record
- scoring summary
- transcript comparison notes when relevant
- defect log
- final decision
