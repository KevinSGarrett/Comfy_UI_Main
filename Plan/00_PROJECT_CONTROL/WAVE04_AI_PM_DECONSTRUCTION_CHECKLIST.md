# Wave 04 — AI Project Manager Deconstruction Checklist

The AI project manager must use this checklist before moving to Wave 05.

## Required confirmations

- [ ] Confirm the Main Flow is treated as a source/staging canvas.
- [ ] Confirm the Main Flow is not treated as final architecture.
- [ ] Confirm every SaveImage terminal lane has been inventoried.
- [ ] Confirm every Note node is classified as a boundary/documentation item unless converted into executable work later.
- [ ] Confirm all ordinary `LoraLoader` catalog nodes remain disabled/disconnected.
- [ ] Confirm the disabled LoRA library is scheduled for registry extraction.
- [ ] Confirm active stack compatibility must be verified before runtime promotion.
- [ ] Confirm fixed LoadImage and LoadImageMask nodes are not acceptable for autonomous production pass chaining.
- [ ] Confirm IPAdapter and ControlNet lanes are staged and need module extraction.
- [ ] Confirm video/audio handoff remains a future-wave boundary.
- [ ] Confirm file-decode QA is not creative QA.
- [ ] Confirm future module extraction must produce API workflow templates, not only UI canvas groups.

## Blocking rules

Do not proceed to implementation extraction if:

- Any SaveImage lane is missing from the inventory.
- Any note boundary is treated as implemented proof.
- Any disabled LoRA catalog node is promoted into production graph without registry-based selection.
- Any engine/LoRA compatibility mismatch is ignored.
- Any fixed image/mask lane is promoted as autonomous production.
