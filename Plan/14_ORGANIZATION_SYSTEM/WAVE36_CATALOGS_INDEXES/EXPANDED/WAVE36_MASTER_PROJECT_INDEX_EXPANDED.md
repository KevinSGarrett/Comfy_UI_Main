# Wave 36 Master Project Index Expanded

The master index is the top-level map of the system.

## It must answer

- What is the canonical structure?
- Where are workflows?
- Where are schemas?
- Where are registries?
- Where are scripts?
- Where are generated outputs?
- Where are proof files?
- Where are release manifests?
- Which items are canonical?
- Which items are examples?
- Which items are archived?
- Which items are stale or missing proof?

```text
project_summary
top_level_directories
catalogs
canonical_workflows
active_registries
validation_scripts
release_artifacts
proof_boundaries
stale_index_status
```
