# Wave 32 Successful-Run Learning Architecture

Successful-run learning captures reusable patterns.

## Positive learning
- model stack worked
- prompt pattern worked
- mask ownership succeeded
- low-denoise pass preserved identity
- audio/video sync worked
- variant selection worked

## Negative learning
- failure mode repeated
- rerun scope was too large
- wrong engine/model family was used
- mask was too broad
- contact/occlusion state failed
- temporal/audio state drifted

## Rule
Learning records are advisory. They do not bypass QA gates.
